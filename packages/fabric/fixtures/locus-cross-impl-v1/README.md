## Conformance

A conforming Content Fabric v1 implementation must:

- parse `pack.json`, `keyring.json`, and `catalog.json` as UTF-8 JSON with integer-only number handling;
- treat `pack.createdAt` as an RFC 3339 UTC string, matching the current Locus v1 validator and docs;
- derive `keyId = sha256(raw 32-byte Ed25519 public key)`;
- canonicalize signing payloads as JCS over the object with top-level `signatures: []`;
- reproduce the three committed `expected-*.jcs` byte-for-byte;
- verify the committed keyring root signature, pack publisher signature, and catalog root signature;
- compute `7cb83d246d67fa50abea10b3fd1a21d97c690bcd2d16ad28cef5c26f6cc27945` as the SHA-256 of `payload/hello.txt`;
- compute `sha256:7820e0546b4affc57c54094e63611dae3195756ff7fd02b64d3a67e3b9e96533` as the signed pack manifest digest;
- compute `d0ace1a6cacd48784011517fe1d21c6c1bf18d7d` as the deterministic BitTorrent v1 infohash for `payload/hello.txt`.

Payload bytes are exactly the UTF-8 string:
